# Project Structure

The repository is split by responsibility: C benchmark implementation, scripts,
matrix metadata, results, and Python analysis utilities.

```text
.
├── config/
│   └── matrix_sources.tsv
├── docs/
│   ├── docs/
│   └── mkdocs.yml
├── figures/
├── include/
├── matrices/
├── results/
│   ├── arm_results/
│   └── x86_results/
├── scripts/
├── src/
├── utils/
│   ├── data/
│   └── plotting/
└── Makefile
```

## Important Directories

`src/`
: C implementation of parsing, CSR/COO conversion, SpMV kernels, reorderings,
benchmark execution, timers, and metrics.

`include/`
: Header files for the C modules.

`config/`
: Project-level configuration and input metadata. The matrix download manifest
lives here as `matrix_sources.tsv`.

`scripts/`
: Executable helper scripts. Matrix downloading and full benchmark execution
are launched from here.

`matrices/`
: Downloaded Matrix Market files. The benchmark expects this structure:

```text
matrices/<category>/<matrix>.mtx
```

`results/`
: CSV output from benchmark runs. The C code writes into the platform-specific
subdirectory:

```text
results/x86_results/
results/arm_results/
```

`utils/data/`
: Python data loading and plotting configuration helpers.

`utils/plotting/`
: Python modules that generate figures and tables. Plot code is split by output
type:

```text
utils/plotting/heatmaps.py
utils/plotting/barcharts.py
utils/plotting/faceted.py
utils/plotting/sparsity.py
utils/plotting/tables.py
```

## Generated Content

The following directories are generated or regenerated by commands:

- `bin/`: compiled benchmark binaries.
- `results/`: benchmark CSV output.
- `figures/`: Python-generated plots and LaTeX tables.
- `docs/site/`: MkDocs static site output.
